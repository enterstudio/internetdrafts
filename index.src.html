Introduction

This section is not normative. Cookies, as defined by [[RFC6265]], diverge from the web's general security policy in a number of ways which may be surprising to implementers and authors who haven't carefully read that document's discussion of [domain matching](http://tools.ietf.org/html/rfc6265#section-5.1.3) and [path matching](http://tools.ietf.org/html/rfc6265#section-5.1.4), or who ignored the ["Weak Confidentiality"](http://tools.ietf.org/html/rfc6265#section-8.5) and ["Weak Integrity"](http://tools.ietf.org/html/rfc6265#section-8.6) sections. This document describes a mechanism by which servers can opt-in to harmonizing cookies' security policy with the [same origin policy](https://tools.ietf.org/html/rfc6454#section-3) [[!RFC6456]] that web authors know and love. User agents that support these "origin cookies" will ignore a Set-Cookie header's value's Path, Domain, and Secure attributes if a new Origin attribute is present, instead tying the cookie to the origin. Origin cookies will be returned in a new Origin-Cookie header field, as the Cookie header field does not provide any information about the source of the cookie.

Examples

The server can set an origin cookie, and can expect it to be returned via the Origin-Cookie header field on subsequent requests. Origin cookies support all the attributes of legacy cookies (as defined by [[RFC6265]]), with the exception of Path, Domain, and Secure; these are ignored.
Origin cookies are set via the Origin attribute in the Set-Cookie header field. That is, given a server's response to a user agent which contains the following header field: 
      Set-Cookie: SID=31d4d96e407aad42; Origin
Subsequent requests from that user agent can be expected to contain the following header field: 
      Origin-Cookie: SID=31d4d96e407aad42
Non-origin cookies are returned in the Cookie header field as usual. If both non-origin and origin cookies are present for an origin, then both a Cookie and Origin-Cookie header field will be present. That is, given a server's response to a user agent which contains the following header fields: 
      Set-Cookie: SID=31d4d96e407aad42; Origin
      Set-Cookie: lang=en-US; Path=/; Domain=example.com
Subsequent requests from that user agent can be expected to contain the following header fields: 
      Cookie: lang=en-US
      Origin-Cookie: SID=31d4d96e407aad42
